### Slide 7: ⭐ SA-AGD 核心创新 (上) ⏱ 1:30-1:50

```
┌────────────────────────────────────────────────────────────┐
│         ✨ SA-AGD: Semantic-Aware Adaptive Gaussian          │
│                  Densification (核心创新)                     │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  问题: 传统3DGS仅用几何梯度 → 物体边界精度不足              │
│  方案: 增加语义边界信号作为第二路径                          │
│                                                            │
│  ┌─────────────────────────────────────────────────────┐  │
│  │              SA-AGD 双路径架构                       │  │
│  │                                                     │  │
│  │  ┌─────────────────┐     ┌─────────────────┐        │  │
│  │  │   几何路径 (Geom)│     │  语义路径 (Sem)  │        │  │
│  │  │                  │     │                  │        │  │
│  │  │ projection_      │     │ KNN spatial      │        │
│  │  │ coverage()       │     │ neighbors (k=8)  │        │  │
│  │  │   ↓              │     │   ↓              │        │  │
│  │  │ 2D投影面积        │     │ torch.cdist       │        │  │
│  │  │ + 深度加权        │     │ 邻域特征方差      │        │  │
│  │  │   ↓              │     │   ↓              │        │  │
│  │  │ geom_importance  │     │ sem_boundary_    │        │  │
│  │  │ ∈ [0,1]          │     │ score ∈ [0,1]    │        │  │
│  │  └────────┬─────────┘     └────────┬─────────┘        │  │
│  │           │                        │                   │  │
│  │           └──────────┬─────────────┘                   │  │
│  │                      ▼                                 │  │
│  │  ┌──────────────────────────────────────┐             │  │
│  │  │         should_densify()              │             │  │
│  │  │                                      │             │  │
│  │  │  clone ← geom>θ_geom ∨ sem>θ_sem     │             │  │
│  │  │  split ← geom>θ_geom ∧ scale>θ_s     │             │  │
│  │  │  prune ← opacity < θ_opac            │             │  │
│  │  └──────────────────────────────────────┘             │  │
│  └─────────────────────────────────────────────────────┘  │
│                                                            │
│  关键参数: sem_grad_weight α (默认0.3)                      │
│    α=0.0 → 纯几何 (3DGS标准)                                │
│    α=0.3 → SA-AGD (推荐, 平衡精度与性能)                     │
│    α=0.6 → 过度增强 (更多语义clone)                          │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

**演讲要点**:
- "SA-AGD的核心思想: 两条路径各自计算评分，然后融合决策"
- "几何路径: 投影覆盖面积越大→越需要densify"
- "语义路径: 邻域语义特征方差越大→越接近边界→额外clone"
- "关键超参数α控制两条路径的权重"

**代码对应**: `adaptive_density_cuda.py:90-166` (compute_semantic_boundary_score + compute_geometric_importance)

**展示建议**: 屏幕上同时打开 `adaptive_density_cuda.py` 滚动到关键函数

### Slide 8: ⭐ SA-AGD 核心代码 (下) ⏱ 1:50-2:10

```
┌────────────────────────────────────────────────────────────┐
│              SA-AGD 关键代码实现                             │
├────────────────────────────────────────────────────────────┤
│                                                            │
│ ① GPU KNN 语义边界评分:                                     │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ def compute_semantic_boundary_score(xyz, sem, k=8):  │  │
│  │     # 批量距离计算 (O(N²)→GPU并行)                    │  │
│  │     dists = torch.cdist(xyz, xyz)  # [N, N]          │  │
│  │     # 取k近邻                                         │  │
│  │     _, knn_idx = dists.topk(k+1, largest=False)      │  │
│  │     # 邻域语义特征方差 → 边界评分                      │  │
│  │     knn_sem = sem[knn_idx[:, 1:]]  # [N, k, D]       │  │
│  │     boundary_score = knn_sem.var(dim=1).mean(dim=1)  │  │
│  │     return boundary_score  # [N] ∈ [0,1]             │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                            │
│ ② 双路径融合决策:                                           │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ def should_densify(geom_score, sem_score, α=0.3):    │  │
│  │     # 几何克隆                                         │  │
│  │     clone_geom = geom_score > self.grad_threshold     │  │
│  │     split_geom = clone_geom & (scales > scale_th)     │  │
│  │     # 语义边界额外克隆                                  │  │
│  │     clone_sem = sem_score > self.sem_boundary_thresh  │  │
│  │     # 融合: 语义路径增加额外clone                       │  │

│  └──────────────────────────────────────────────────────┘  │
│                                                            │
│ ③ 向量化执行 (GPU批量):                                     │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ # Clone: 复制xyz + 小幅扰动 → GPU批量cat              │  │
│  │ # Split: 沿最大scale轴分裂 → 两倍高斯                  │  │
```

## 五、答辩 Q&A 预判

| # | 可能问题 | 建议回答 |
|---|---------|---------|
| 1 | "SA-AGD和传统3DGS密度控制的本质区别是什么?" | 传统仅使用几何梯度(单路径), SA-AGD增加了语义边界信号(第二路径)。两个信号通过OR逻辑融合: 任一触发就增加密度 → 物体边界处更密集。 |
| 2 | "为什么用K-means而不是真实SAM/CLIP?" | 当前阶段使用K-means作为语义特征的代理验证SA-AGD概念。K-means生成的正交伪特征足以模拟不同语义区域,且不需要额外模型加载和推理开销。集成真实VFM是后续工作。 |
| 3 | "Chamfer距离改善24.8%是否显著?" | 是。在400→500高斯的微小增量下, Chamfer的24.8%改善表明语义增强有效提升了边界区域的重建精度,且高斯数增长可控(1.25x)。 |
| 4 | "渲染为什么有两种模式?" | Tile-based用于快速可视化(benchmark),不可微但更快。Batch splat用于训练,保持完整autograd图。两者根据`differentiable`参数自动切换。 |
| 5 | "RTX 3060上真的能达到实时吗?" | 对于1K高斯: ~8ms(125 FPS)远超实时。对于100K高斯: ~5ms(200 FPS)通过tile-based路径。VRAM占用<200MB,远低于8GB。 |
| 6 | "和现有SLAM系统(如ORB-SLAM3)相比如何?" | 我们的系统是学术Demo级(概念验证),聚焦于验证SA-AGD创新的有效性,而非和成熟SLAM系统比ATE。真实数据集评估是后续工作。 |
| 7 | "因子图中的视觉因子为什么是identity矩阵?" | 当前版本使用匹配分数加权的identity矩阵作为简化Hessian,完整实现需要从点图对齐残差累积真实的视觉因子。这是P1待完成项。 |
